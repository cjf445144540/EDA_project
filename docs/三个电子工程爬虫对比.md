# 三家电子工程爬虫技术对比（电子工程网 / 电子工程专辑 / 电子工程世界）

## 1. 共同点
- 都采用 Selenium + Headless Chrome，`page_load_strategy='eager'`，并带有 SSL/证书忽略与 UA 伪装配置。
- 都支持共享 `driver`（`shared_driver`）和独立 `driver` 两种模式。
- 都有分页抓取 + 日期截断（`cutoff_date`）机制，遇到超出日期范围会提前停止。
- 都在抓取后做链接去重（按 `link` 去重）。
- 都在 `crawl()` 中支持 `min_content_length` 字数阈值过滤（当前默认 500），不足阈值会在详情页二次抓正文后再判断。
- 三家 `fetch_news_content()` 均已改为 `requests` 优先，失败再回退 Selenium。

## 2. 关键差异
| 维度 | 电子工程网（eechina） | 电子工程专辑（eetchina） | 电子工程世界（eeworld） |
|------|------------------------|---------------------------|---------------------------|
| 搜索入口 | `https://www.eechina.com/search.php` | `https://www.eet-china.com/e/sch/index.php` | `https://so.eeworld.com.cn/s` |
| 列表定位 | `li a[href*="thread-"] / viewthread / .xst` | `ul.main_list > li.search-article-item` | `table.result > tr` |
| 日期提取 | `cite` 文本正则 + 行文本兜底 | `span.search-article-item-date` 正则提取 | `a[s_pub]` 优先，`span.g` 兜底 |
| 页面重试策略 | 共享driver时 2 次重试，独立driver时 3 次，并带 fallback 超时等待 | 共享driver时 2 次重试，独立driver时 3 次 | 共享driver时 2 次重试，独立driver时 3 次 |
| 详情正文核心选择器 | `td[id^=postmessage_]` / `td.portal_content` / `.t_f` | `.article-con` / `.article-detail` / `.article-body` | `.newscc` / `article .newscc` / `.article-content` |
| 列表阶段是否预存 content | 否 | 否 | 是（从搜索结果摘要预提取 `content`，后续可复用） |
| 列表抓取通道 | Selenium | Selenium | requests 优先，失败回退 Selenium |
| 性能特征 | WAF/论坛结构导致等待与稳定性波动较大 | 页面结构相对规范，但详情页正文区域多样 | 已做“双通道+耗时统计+摘要复用”优化，整体速度上限最高 |

## 3. 当前统一策略（run_crawler 主流程）
- 三家配置统一使用：`DEFAULT_DAYS`、`DEFAULT_MAX_PAGES`、`DEFAULT_MIN_CONTENT_LENGTH`。
- 三家都走 Selenium 顺序来源分组（与腾讯/搜狐同组）。
- 最终列表阶段统一最小正文长度阈值为 200（共享规则），三家在各自 `crawl()` 内先做 500 的主过滤。

## 4. 电子工程世界最新加速点
- 列表阶段增加 requests 快速通道，只有失败才回退 Selenium。
- 新增列表解析复用函数，避免重复解析逻辑。
- 每页输出通道与耗时：`requests/selenium + 秒数`，便于定位慢点。
- 正文过滤阶段输出命中统计：`requests命中` 与 `selenium回退命中`。
- 正文抓取支持复用外部 fallback driver，减少重复创建/销毁 Chrome。

## 5. 电子工程网最新加速点
- 列表阶段增加 requests 快速通道，只有遇到反爬/结构不匹配时才回退 Selenium。
- 将搜索页 URL 生成与列表解析逻辑函数化，减少重复解析开销。
- 每页输出通道与耗时：`requests/selenium + 秒数`，便于快速判断瓶颈来源。
- 正文过滤阶段输出命中统计：`requests命中` 与 `selenium回退命中`。
- 正文抓取支持复用共享 driver（仅共享模式复用），避免重复启动 Chrome。

## 6. 调优建议
- 速度优先：优先减少 `max_pages`，其次再调 `days`。
- 稳定优先：保留共享 driver 的顺序执行，避免多 driver 并发触发反爬；保留 requests->Selenium 回退链路。
- 质量优先：维持 `min_content_length=500`，但可对特定来源单独降低到 300 作为折中。

## 7. 速度差异补充（为什么电子工程专辑常更快）
- 列表结构更规整：电子工程专辑列表节点稳定（`ul.main_list > li.search-article-item`），解析分支少、异常少。
- 日期提取更直接：专辑主要从 `span.search-article-item-date` 提取日期，兜底逻辑较少，CPU 与判定开销更低。
- 回退链路触发更少：电子工程网/电子工程世界更容易遇到结构波动或反爬，触发“慢路径”回退（requests→Selenium 或额外重试）。
- 单页有效新闻成本更低：专辑列表噪音少、有效条目占比高，同样 1 页下通常更快完成。
- 结论：专辑快的核心不是“线程更多”，而是**页面稳定性更高 + 解析路径更短 + 回退概率更低**。
